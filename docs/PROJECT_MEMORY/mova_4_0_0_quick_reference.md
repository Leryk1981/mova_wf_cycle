# MOVA 4.0.0 — короткий довідник для MOVA Skills Lab

## 1. Визначення

**MOVA (Machine-Operable Verbal Actions)** — це мова машинно-виконуваних домовленостей про дані й дії.  
Вона описує, *які саме* структуровані дані допускає система та *які мовні акти* з цими даними можна виконувати, але сама нічого не виконує.

MOVA складається з чотирьох базових блоків:

- **Data Schemas (`ds.*`)** — JSON-схеми, які визначають форму та зміст даних (профіль, конфігурація, результат, епізод тощо).
- **Envelopes (`env.*`)** — мовні акти над даними: «запусти», «перевір», «запиши епізод». Envelope завжди посилається на одну чи кілька `ds.*`-схем.
- **Global** (глобальні словники та узгоджений простір імен) — фіксує спільні для всіх контрактів ролі, ресурси, стани, ідентифікатори версій, а також правила іменування (`ds.*`, `env.*`, `episode.*`).
- **Episodes (`ds.episode_*`)** — записи про факти виконання: хто що запустив, з якими даними, який результат і які метрики отримано. Епізоди належать до «генетичного» шару й описують історію роботи системи.

Усі ці блоки — *контракти*: вони визначають, що вважається коректними даними, дією чи епізодом, але не містять виконуваного коду.

## 2. MOVA у контексті MOVA Skills Lab

MOVA Skills Lab спирається на MOVA 4.0.0 як на червоне ядро контрактів і додає до нього:

- **жовтий шар (skills/)** — виконавчі навички, які:
  - приймають envelopes (`env.*`) як вхід;
  - працюють з даними, описаними через `ds.*`;
  - за потреби створюють епізоди (`ds.episode_*`) для генетичного шару;
- **зелений шар (lab/)** — приклади envelope, реєстр skills, утиліти запуску, тимчасові артефакти, епізоди.

Усі нові скіли та епізоди в цьому репозиторії мають бути узгоджені з версією MOVA, вказаною у відповідних схемах (`"mova_version": "4.0.0"`).

## 3. Канонічна спекація

Цей репозиторій **не є** джерелом правди для схем MOVA.  
Канонічна спекація MOVA 4.0.0 публікується окремо:

- **Canonical spec:** `https://github.com/Leryk1981/mova-4.0.0`

У цьому репозиторії ми лише:

- посилаємося на MOVA 4.0.0 як на базову версію;
- визначаємо додаткові контракти `ds.*`, `env.*` та `ds.episode_*`, специфічні для Skills Lab;
- реалізуємо жовті skills, які працюють *поверх* цих контрактів.

Якщо текст цього довідника колись розходиться зі спекацією `mova-4.0.0`, джерелом правди вважається канонічний репозиторій MOVA.
